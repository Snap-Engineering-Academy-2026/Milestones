# Arrow functions, promises and async/await

Below are some practice problems and learning resources for these funky topics!

## Practice Problems

1. ["Coding Meetup" Arrow Functions series](https://www.codewars.com/collections/coding-meetup-series) - List of problems to practice using arrow functions with array builtins like [`.filter()`](https://www.codewars.com/kata/583a8bde28019d615a000035) and [`.map()`](https://www.codewars.com/kata/58279e13c983ca4a2a00002a). I also really like [this one as it's a bit more layered and requires more algorithmic thinking](https://www.codewars.com/kata/583d972b8bbc0402cf000121). 
2. [Jokes you've been 'awaiting' for ... promise](https://www.codewars.com/kata/5a353a478f27f244a1000076) - Writing your own async function: Promises, async/await, fetch()
3. [Promises Made and Broken: The Misadventures of Bob the Highly Paid Consultant](https://www.codewars.com/kata/587593285448632b8d000143/train/javascript) - Fix broken code written by "Bob" the consultant, Asynchronous programming concepts and promises

## Learning Arrow Functions

1. [Short and sweet intro to Arrow Functions](https://www.learn-js.org/en/Arrow_Functions)
2. [Slightly longer "Arrow Functions Basics"](https://javascript.info/arrow-functions-basics) and [Arrow Functions Resvisited](https://javascript.info/arrow-functions)

## Learning Promises

1. [Short-ish and sweet - Promise Basics](https://javascript.info/promise-basics) - explains promises along with `.then()` syntax.
2. [Interactive Promises tutorial on CodingGame](https://www.codingame.com/playgrounds/347) (website seems to be temporarily broken as of July 7th 2025) - [partially working archive link](https://web.archive.org/web/20231129053449/https://www.codingame.com/playgrounds/347/javascript-promises-mastering-the-asynchronous/what-is-asynchronous-in-javascript)
3. [In depth interactive tutorial on fetch()](https://www.codecademy.com/courses/learn-javascript-requests) - "In this lesson, we will explain how to use fetch() and promises to handle requests. Then, we will simplify requests using async and await." Free account needed.
